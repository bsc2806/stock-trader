## 2026-07-23 · 전체 코드 감사 + 정산/글리치 버그 수정

모의 2~4주 검증을 에러 없이 돌리기 위해 전체 코드를 점검하고 다음을 수정.

### 🐞 [치명적] 가짜 고점으로 전량청산 오작동
- **증상**: 계좌가 +9.78% 수익인데도 대시보드에 낙폭 -36.3% · 전량청산(LIQUIDATE) 단계로 표시. 방치 시 다음 개장에 전 종목 강제 청산될 뻔.
- **원인**: 첫날 초과매수(미수)+정산지연 때 KIS `nass_amt`(순자산)가 일시적으로 17M로 잘못 보고됨(주식 풀평가 + 아직 안 줄어든 현금 중복 계산) → 그 값이 고점(peak)으로 저장됨.
- **수정**:
  - 순자산을 KIS `nass_amt` 대신 **(정산현금 + 보유평가액)로 직접 계산** (`kis_client.get_balance`) → 글리치 원천 차단.
  - 잘못 저장된 고점(17,236,294) → 현재 순자산으로 리셋.
  - 트레이더: **순자산 조회 이상(0 이하)이면 매매 보류** 가드 추가(글리치 시 가짜 청산 방지).

### 🐞 [높음] 초과매수(미수) 발생
- **증상**: 첫날 매수 총액 11.2M로 계좌(10M) 초과 → 예수금 마이너스(미수).
- **원인**: 보유주식 평가액을 `순자산 - 예수금`으로 계산했는데, 예수금이 D+2 정산 전이라 매수가 즉시 반영 안 돼 보유액이 0으로 잡힘 → 예산 과다 산정.
- **수정**: 보유평가액 = **보유 종목 평가액 직접 합산**(정산지연 무관)으로 예산 계산.

### 💰 예수금 정산 반영 + 정확한 주문가능금액
- **예수금 표시**: 원시 예수금(`dnca_tot_amt`)은 매도대금 D+2 정산 전이라 음수로 보임 → **D+2 정산 반영액(`prvs_rcdl_excc_amt`)을 실제 가용 현금**으로 사용. 대시보드엔 **정산반영 + 정산전 둘 다 표시**(차이=정산 대기 매도대금).
- **예산 기준**: 잔고 예수금(근사치) 대신 **매수가능조회 API(`inquire-psbl-order`)의 미수없는매수금액(`nrcvb_buy_amt`)** 사용 → 미수·초과매수·주문거부 원천 방지. (미수 포함 최대금액은 절대 미사용.)
- 매도대금은 **당일 즉시 재매수 가능**(2일 대기 아님).

### 🔧 코드 감사 수정
- **DART 장애 격리**: 상장폐지 조회(`get_corp_map`) 실패 시 전체 사이클이 중단돼 손절/트레일링까지 멈추던 문제 → 상폐체크만 생략하고 **손절/트레일링은 계속 작동**.
- **파일 쓰기 경합 제거**: `state.json`·`positions.json`을 서버·트레이더가 동시에 쓰던 문제 → **서버는 읽기 전용**, 쓰기는 트레이더 전담(peak/포지션 데이터 손상 방지).
- **토큰 캐시 견고화**: `token_cache.json` 동시 접근으로 손상돼도 요청이 죽지 않고 **새로 발급**.

## 2026-07-22 · 트레일링 청산 도입

- **청산 전략**: 고정 목표 대기 → **트레일링 스탑**(`risk.exit_decision`). 손실은 짧게 손절, 이익은 고점 추적 후 반납 시 익절. 근거: 한국 매도세(~0.18%)로 잦은 소액매매 불리 → "손실 짧게, 이익 길게".
- 종목별 최고 수익률(peak)을 `positions.json`에 추적. 단기/장기 각각 손절·발동·반납·상한 기준 분리.

## 2026-07-22 · 계좌 금액 자동 적응 + 동전주 방지

- **자동 사이징**: `affordable_slots`(예산/최소주문액=종목수) + `size_each`(소액이면 종목당%캡 대신 최소주문액 허용). 50만~1천만 어느 금액이든 설정 변경 없이 자동 적응.
- **동전주 차단**: `min_share_price` 미만 주가 매수 금지(실전 2,000/모의 500원). 주가>종목당예산이면 자동 건너뜀.

## 2026-07-20 · 자동화 + 안정화

- **자동 실행**: Windows 작업 스케줄러 `StockTrader`(평일 09:00~15:30, 5분마다 `run_trader.bat`).
- **실행 로그**: 트레이더 활동/오류를 `data/trader_log.json`에 기록 → 대시보드 "실행 로그" 탭(오류 빨강).
- **단계적 낙폭 방어**: 전량청산 한 방 대신 -12% 매수중지 → -20% 부분정리 → -30% 전량청산.
- **비용 최소화**: AI 종목선정을 결정주기(기본 240분)마다 1회+정규장에만(`plan_store` 공유). 장 마감/주말 호출 0.
- **KIS 500 오류**: `_get_with_retry`(5xx 재시도) + 서버 last-known 캐시 → 대시보드에 오류 미표시.
