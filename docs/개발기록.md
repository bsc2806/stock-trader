# 개발 기록 (상황별)

프로젝트가 만들어진 과정을 상황·결정·근거 순으로 기록. (2026-07)

---

## 0. 최초 구상
- **상황**: "AI가 주석(공시) 정보를 분석해 실제 보유 자금으로 투자하는 프로그램"을 만들고 싶다.
- **필요 요소 정리**: 증권사 API + 계좌 + 투자 판단용 데이터.

## 1. 데이터·증권사 선택
- **결정**: 데이터는 **DART 공시(1차) + 뉴스(보조)**. '카더라'는 참고만.
- **결정**: 증권사는 **한국투자증권(KIS)**. REST API라 윈도우에서 다루기 쉬움.
- **결정**: **모의투자 우선** → 검증 후 소액 실전.
- **근거**: 공시는 팩트 기반이라 AI 분석 신호로 적합. 모의로 로직 검증 후 실전이 안전.

## 2. Phase 1 뼈대 (읽기 전용)
- DART 공시 수집 → AI 분석 → 로그 출력. 주문 코드 없음(안전).
- 보안: 키는 `.env`에만, `.gitignore`로 유출 차단. **키는 채팅에 붙여넣지 않고 사용자가 .env에 직접 입력.**
- 모듈: `config / dart_client / kis_client / analyzer / main`.

## 3. 환경 설정
- **상황**: 로컬에 Python 미설치.
- **처리**: winget으로 Python 3.12 설치, 프로젝트 `.venv` 생성, 의존성 설치.
- **참고**: Bash엔 python PATH 없음 → PowerShell + `.\.venv\Scripts\python.exe` 사용.

## 4. 키 발급 이슈
- DART 인증키는 이메일이 아니라 **OpenDART '인증키 관리'의 40자리 문자열**. '승인' 상태면 발급 완료, IP는 선택.
- KIS는 실전용/모의용 앱키가 다름 → **모의용**으로 발급.
- **KIS·DART 실동작 확인**: 모의 예수금 1천만원, 공시 수집 정상.

## 5. 실시간 대시보드
- Flask 웹서버(`server.py`) + `web/index.html`. localhost:5000.
- 손익 히어로 + 보유 + 관심종목/AI신호. 이후 탭 구조로 개편.
- **KIS 간헐 500 오류** → `_get_with_retry`(5xx 재시도) + 서버 last-known 캐시로 해결. 대시보드엔 오류 미표시.
- 장 마감 시 자동 새로고침 중지 + 수동 버튼(비용·자원 절약).

## 6. AI 연동 (크레딧)
- **상황**: AI 호출이 400(invalid_request)로 실패.
- **원인**: 코드 문제 아님 — Anthropic 계정 **크레딧 부족**. 충전 후 정상.
- 구조화 출력(JSON schema)로 신호/근거를 안정적으로 수신.

## 7. 방향 전환 — 자율 트레이더
- **상황(사용자)**: 고정 관심종목이 아니라 **AI가 스스로 종목을 발굴·선정·매매**해야 한다. 실전도 완전 자동 원함.
- **최우선 제약(사용자)**: "원금 잃으면 죽음" — 개별 손실은 OK, 전체 원금 손실은 불가.
- **대응**: 자본 방어 규칙을 1순위로. 시장 스캔(`scanner`) + AI 선정(`select_portfolio`) + 리스크 모듈(`risk`) 도입.
- **솔직한 고지**: 손실 0 보장 불가 → 모의 검증 → 소액 실전 → 원금은 자동매매 배제 권고.

## 8. 자율 매매 실행 (모의)
- `trader.py`: 스캔→AI선정→리스크→모의 주문→청산. `kis_client.place_order`(시장가·해시키·재시도 없음=중복방지).
- 안전 순서: 실전차단 → 장시간 가드 → 낙폭 방어 → 손절/익절 → 당일 한도 → 리스크 통과분만 매수.
- **주문 배선 검증**: 토요일이라 KIS가 "모의투자 영업일이 아닙니다"로 거부 = 정상. 월요일 체결 예정.

## 9. 상장폐지 회피 (사용자 요청)
- **요청**: 상폐될 만한 종목은 매수 배제, 보유 중 상폐 조짐이면 손실 최소화 매도.
- **처리**: `analyzer.delisting_risk`(관리종목·실질심사·자본잠식·횡령/배임·거래정지 등). 스캔 제외 + AI 회피 + 보유 청산.

## 10. 적극 모드 + 장기/단기 (사용자 요청)
- **요청**: 모의에선 다양하게 적극 시도하고, 아닌 건 스스로 습득해 제한.
- **처리**: 모의=적극 프로파일(주식80%·다종목), AI 선정도 적극 모드(명백한 위험만 배제).
- **요청**: AI 계획을 장기/단기로 구분, 보유기간(몇 개월/며칠) 명시.
- **처리**: `select_portfolio`가 horizon+holding_period 반환. 스캔도 `horizon_hint`로 장기/단기 태그. 매수 종목 수 축소(10→6).

## 11. 비용 최소화 (사용자 우려)
- **우려**: 새로고침마다 AI 호출되면 크레딧 낭비?
- **확인/처리**: 새로고침은 AI 미호출(캐시). AI 선정은 `plan_store`+`AI_DECISION_INTERVAL_MIN`(기본 240분)으로 주기당 1회·정규장에만. 서버·트레이더 공유. 장 마감/주말 호출 0. 공시 캐시로 재분석 0.

## 12. 서킷브레이커 → 단계적 방어 (사용자 지적)
- **지적**: 계좌 -20%에 전량청산은 너무 뭉툭 — 회복 가능한 조정에서 바닥 매도할 수 있음.
- **판단·결정**: 개별 손절은 유지, 계좌 방어는 **단계적**으로 — -12% 매수중지 / -20% 손실종목 절반정리 / -30% 전량청산.
- **근거**: 개별 종목 붕괴는 손절이, 시장 급락은 단계적 방어가 담당. 대형주 시장 조정은 대개 회복.

## 13. 수익 기준 → 트레일링 스탑 (사용자 위임)
- **질문**: 고정 목표 대기 말고, 자주 매매해서라도 포트폴리오 전체가 플러스 되게?
- **판단(내 추천)**: 한국은 매도세(~0.18%)+수수료·슬리피지로 **잦은 소액매매가 불리**. 정반대로 **손실 짧게(손절), 이익 길게(트레일링)**가 효과적.
- **처리**: `risk.exit_decision(pnl, peak, horizon)` — STOP/TRAIL/TAKE/HOLD. peak(최고수익률)를 `positions.json`에 추적. 단기 손절-5/발동+6/반납3/상한+20, 장기 손절-14/발동+12/반납6/상한+40.
- **솔직한 고지**: 규칙은 확률만 높임, 수익 보장 아님 → 모의 검증 필수.
