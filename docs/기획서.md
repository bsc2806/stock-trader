# AI 자율 주식 트레이더 — 기획서

## 1. 프로젝트 개요

AI가 **뉴스·공시 정보를 스스로 분석**해 종목을 발굴·선정하고, **매수·매도·시점을 자율 판단**해
실제 보유 자금으로 투자하는 개인용 자동매매 프로그램.

- **핵심 차별점**: 고정된 관심종목이 아니라, AI가 시장 전체를 스캔해 스스로 종목을 고름.
- **진행 원칙**: 모의투자로 충분히 검증 → 소액 실전 → (검증 후) 확대.
- **개발 시작**: 2026-07 (개인 프로젝트)

## 2. 핵심 원칙

1. **원금 보존이 최우선.** 개별 손실은 감수하되, 전체 원금을 한 번에 잃는 경로를 구조적으로 차단.
2. **모의 우선.** 실전 주문은 명시적 플래그(`LIVE_TRADING=true` + `KIS_ENV=prod`)가 있어야만 동작. 기본은 모의 전용.
3. **신뢰 소스만.** DART 공시를 1차 신호, 뉴스를 보조로. '카더라'식 정보는 매매에 미반영.
4. **AI는 판단, 실행은 규칙이 감쌈.** LLM이 종목/방향을 결정하되, 실제 주문은 리스크 규칙을 통과한 것만.
5. **비용 최소화.** AI 호출은 결정 주기마다 1회로 제한, 보호 규칙은 무비용 계산.

> ⚠️ **한계 고지**: 어떤 자동매매도 손실을 0으로 만들 수 없다. 규칙은 손실 확률을 낮추고
> 이익 확률을 높일 뿐 수익을 보장하지 않는다. 실전은 반드시 '잃어도 되는 소액'으로만.

## 3. 시스템 구조

```
[시장 스캔] 전체 코스피 공시(주요사항·거래소) 훑어 신호성 종목 발굴
     │        (상장폐지 위험 종목 제외, 노이즈 필터)
     ▼
[AI 선정] Claude가 후보 중 매수 종목 선정 (장기/단기 구분 + 보유기간 + 근거)
     │
     ▼
[리스크 규칙] 원금 방어선 통과분만 (예산·종목당 한도·낙폭 방어)
     │
     ▼
[모의 주문] KIS 모의계좌 매수/매도 실행 (장 운영시간에만)
     │
     ▼
[청산 관리] 트레일링 익절 / 손절 / 상장폐지 회피 / 단계적 낙폭 방어
     │
     ▼
[기록·대시보드] 체결·포지션·손익 실시간 (localhost:5000)
```

### 모듈 구성 (`src/`)
| 파일 | 역할 |
|---|---|
| `config.py` | `.env` 로딩/검증, 안전 플래그 |
| `kis_client.py` | 한국투자증권 REST: 시세·잔고·주문(시장가, 해시키, 재시도 없음) |
| `dart_client.py` | DART 전자공시: 종목별 공시, corp_code 매핑 |
| `scanner.py` | 시장 전체 공시 스캔 → 신호성 종목 발굴 + 장기/단기 힌트 |
| `analyzer.py` | Claude: 공시 분석, 포트폴리오 선정, 상장폐지 감지, 힌트 |
| `risk.py` | 리스크 규칙: 예산·한도·손절·트레일링·단계적 낙폭 방어 |
| `market.py` | 정규장 운영시간 판단 (평일 09:00~15:30) |
| `plan_store.py` | AI 계획 공유·호출 주기 제한 (비용 절약) |
| `trader.py` | 자율 매매 오케스트레이터 (`--loop` = 정규장 5분마다) |
| `server.py` | 실시간 대시보드 웹서버 (Flask) |
| `autotrade_dryrun.py` | 주문 없이 AI 판단만 출력 (검증용) |

## 4. 데이터 소스

- **한국투자증권 (KIS Developers)** — REST API. 시세·잔고·주문. 모의투자 도메인 사용.
- **DART 전자공시 (Open API)** — 무료. 공시가 팩트 기반이라 1차 신호로 적합.
- **Claude (Anthropic API)** — 공시 분석 및 종목 선정.

## 5. 리스크 관리 (원금 보존)

### 개별 종목 — 손실 짧게, 이익 길게 (트레일링)
한국 시장은 매도 시 거래세(~0.18%)+수수료·슬리피지가 붙어, 잦은 소액 매매는 비용에 잠식된다.
따라서 **손절은 빠르게, 익절은 트레일링으로 크게** 가져간다.

| | 손절 | 트레일링 발동 | 반납 허용 | 상한 익절 |
|---|---|---|---|---|
| 단기 | -5% | +6% | 3% | +20% |
| 장기 | -14% | +12% | 6% | +40% |

- 트레일링: 발동 이후 최고 수익률(peak)을 추적, 고점 대비 반납폭만큼 꺾이면 익절.

### 계좌 전체 — 단계적 낙폭 방어 (한 번에 전량청산 X)
| 낙폭 | 대응 |
|---|---|
| -12% | 신규 매수 중지 |
| -20% | 손실 종목 절반 정리 + 매수 중지 |
| -30% | 전량 청산 (진짜 붕괴에만) |

### 그 외
- **상장폐지 회피**: 관리종목·실질심사·자본잠식·횡령/배임·거래정지 등 감지 → 매수 배제 + 보유 시 즉시 청산.
- **당일 손실 한도**: 당일 -6% 도달 시 신규 매수 중지.
- **투입 상한**: 순자산의 최대 80%(모의)/40%(실전)만 주식, 나머지 현금 방어. 종목당 최대 20%(모의).
- **장 운영시간 가드**: 평일 09:00~15:30에만 주문.

## 6. 매매 프로파일 (모의 vs 실전)
- **모의(적극)**: 다양하게 실험. 주식 80%·최대 6종목·저확신도 시도.
- **실전(보수)**: 원금 보존 우선. 주식 40%·손절 타이트.
- 값은 `risk_config.json`으로 조정 가능.

## 7. 대시보드 (localhost:5000)
탭 구성: **개요**(손익·보유·AI계획) / **포트폴리오**(자산 도넛·통계) /
**AI 발굴·계획**(장기/단기 그룹·근거) / **매매 내역** / **리스크**(방어 사다리·규칙).
- 정규장엔 5초 실시간, 장 마감엔 자동 갱신 중지 + 수동 버튼 (비용·자원 절약).
- 헤더 실시간 시계.

## 8. 비용 관리 (AI 크레딧)
- AI 종목 선정은 **결정 주기(`AI_DECISION_INTERVAL_MIN`, 기본 240분)마다 1회 + 정규장에만**.
- 서버·트레이더가 계획을 공유 → 둘 다 돌아도 주기당 1회로 수렴. 장 마감·주말 호출 0.
- 공시 분석은 접수번호(rcept_no) 캐시로 재분석 방지.
- 대시보드 새로고침은 시세·잔고·캐시(무료)만.

## 9. 로드맵
- **Phase 1** ✅ 읽기 전용 파이프라인 (스캔→분석→로그·대시보드).
- **Phase 2** ✅(구현 완료, 검증 대기) 모의 자동주문 + 리스크·트레일링 청산.
- **Phase 3** ⏳ 소액 실전 (모의 성과 검증 후).
- **다음** 자기학습 루프(매매 성과 피드백), 뉴스 소스, 공시 전문 파싱, 공휴일 달력.

## 10. 실행 방법
```powershell
# 대시보드
python src/server.py            # → http://localhost:5000

# 자율 매매 (정규장 동안)
python src/trader.py --loop

# 검증(주문 없이 AI 판단만)
python src/autotrade_dryrun.py
```
키는 `.env`에만 저장(커밋 금지). 자세한 설치·설정은 `README.md` 참조.
